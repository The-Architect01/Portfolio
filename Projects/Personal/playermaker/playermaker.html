Project Summary

This project originally started as an Excel spreadsheet developed to make formatting D&D player characters. Unfortunately, this spreadsheet quickly got extremely large (>13 mb) and hard to manage. Luckly I was learning how to develop forms using C#.net WinForms in Visual Studio 2019.

Once I developed the framework to build the app (mainly print, auto-updating, and saveing/loading features), I was able to quickly add the races, classes, and backgrounds found in the player's handbook. I am currently planning on adding more sourcebooks and features in the future.

Features

Auto Updating

This feature was the hardest to implement in the way that was originally envisioned. Originally, the plan was to just update the required .resx files which were used to store the data for each of the player options. This was going to be accomplished by using the built in 'Check For Updates' Feature of ClickOnce, the build format that would prepare a .exe for distribution. However, ClickOnce has very precise security standards that were difficult to fullfil on a small scale level. The Solution: make my own auto updating system.

So to make my own custom auto update system, I used the GitHub API and the built in librarys of C#.net.

First, the program checks if a new release has been published on GitHub using the API. If so, download the .zip containing the updated .exe.

Next, check for the zip-slip vulernablility. If the zip file is clean, extract the contents and overwrite the old version.

NOTE: This feature is being reviewed to see how to simplify the process and make it more secure.
