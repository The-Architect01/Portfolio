Project Summary

This game was created for my software development class. In the class, we were tasked to develop a project through the course of an entire semester. To help with such a large project, we were split up into different teams, with this project recieving a total of 5 members. I was elected the leader of the group and lead development of the project.

For our project, we knew that we wanted to create a video game. However, one of the requirements was to make it computer science themed. To do this, we decided on a minigame based educational game that taught players how various data structures operated. We took heavy inspiration from the Mario Party and Survivor franchises in how we envisioned the game play. We decided to attempt to implement the following features:

  • Interactive Minigames
  • Dynamic Menus
  • Menus
  • Multiplayer
  • Gameboard

We divided the group as follows, each person took one goal to work on:

  • Interactive Minigames and Save Data (Me)
  • Dynamic Menus (Bricks Hudson)
  • Menus and Sound (Eric Minic)
  • Multiplayer (Ronit Patel)
  • Gameboard (Tyler Delegrange)

I will be focusing only on the parts that I worked on below, if your are curious about how other aspects of the game work please refer to our Documentation (lovingly called 'The Ancient Text' because of how large it was). Tyler named our project 'Keep Body and Soul Together-er' as a parody of the definition of survive. I am currently working on version 2.0 of the project seen in the splash images.

Features

Arrays Attack!

We Call It Maze

In this minigame a player is navigating a randomly generated binary search tree looking for a specific value displayed as the Goal. The player should complete the search in as little moves as possible and within the given time span of 90 seconds (though it will likely be a lot faster).

The player is allowed to go in three directions; up, left, right. Selecting a direction will consume a move. Up takes the player to the current node’s parent; left sets the current node to its left child; right sets the current node to its right child. The player wins if they select the correct node, and the player loses if they fail to select the correct node in the given time span.

Pointer Panic

In this minigame a player is exploring how pointers are unique memory addresses used by software. The player’s goal is to identify as many ‘pointers’ as they can without making a mistake in the given time span of 90 seconds.

The player clicks on one of three randomly generated ‘pointers’ with the hint ‘pointer’ showing the correct ‘pointer’ with a few distortions. The game randomly generates a different ‘pointer’ for every correct answer and ends if the time runs out or the player clicks the incorrect pointer.

Manic Menus

In this minigame a player is using only Queue behaviors (Dequeue, Enqueue, and FIFO [First In, First Out]) to sort a menu into the correct way as denoted in the Goal Menu. The player is allowed to dequeue items into two temporary cells (representing variables) or into a different menu. For the player to win, either Menu 1 or Menu 2 need to match exactly the Goal Menu in the given time span of 90 seconds.

Robbie's Revenge

In this minigame a player is using their memory to understand the set concept of individuality. The player wants to get as many unique values as they can without repeating themselves or Robbie the robot.

The player must input a value that matches the given category and must not have been previously entered by either the player or Robbie. To win, the player must last the full duration of the time span of 90 seconds without repeating a value. The game is possible to end early if Robbie enters an incorrect entry, determined by Robbie's Difficulty rating.

Towers of Hanoi

In this minigame a player is using only Stack behaviors (Dequeue, Enqueue, and LIFO [Last In, First Out]) to get the starting tower to the furthest right column in the given time span of 90 seconds.

The player is allowed to dequeue items from the top of one stack and enqueue the item into one of the other stacks given that the item is smaller than the one below it. If the item is larger than the item below, the move is canceled.
